# Go Pipeline Migration to Unified Social Integrations Collection

Owner: Bilal Tariq  
Created: 2025-09-08  
Updated: 2025-09-08  
Status: **Migration Complete - Clean Implementation**

## Executive Summary

This document outlines the migration plan for updating the ContentStudio Social Analytics Go pipeline to use the unified `social_integrations` collection with clean, consistent field naming. Since the Go pipeline hasn't been deployed to production, we can implement the ideal structure without backward compatibility concerns.

## Migration Goals

- **Unified Data Access**: Single collection for all social platform accounts
- **Clean Field Structure**: Use `platform_identifier` consistently, no legacy field names
- **Simplified Codebase**: Single repository pattern for all platforms
- **Performance Optimization**: Single query for multi-platform operations
- **Future-Proof Design**: Consistent patterns for adding new platforms

## Components to Update

### 1. MongoDB Models (`/src/pkg/models/mongo/`)

#### Current Structure

- Separate structs: `FacebookAccount`, `InstagramAccount`, `LinkedinAccount`
- Platform-specific fields in each struct
- Individual field names (e.g., `facebook_id`, `instagram_id`)

#### New Unified Structure (Clean Implementation)

```go
// SocialIntegration represents a unified social platform account
type SocialIntegration struct {
    // Core fields (all platforms)
    ID                    primitive.ObjectID `bson:"_id,omitempty"`
    PlatformType         string             `bson:"platform_type"`
    PlatformIdentifier   string             `bson:"platform_identifier"`  // The primary platform ID
    PlatformName         string             `bson:"platform_name"`
    PlatformURL          string             `bson:"platform_url,omitempty"`
    PlatformLogo         string             `bson:"platform_logo,omitempty"`
    Type                 string             `bson:"type"`  // e.g., "Page", "Profile", "Business"
    WorkspaceID          primitive.ObjectID `bson:"workspace_id"`
    UserID               primitive.ObjectID `bson:"user_id"`
    AddedBy              primitive.ObjectID `bson:"added_by"`
    State                string             `bson:"state"`     // "Added", "Deleted", "Disabled"
    Validity             string             `bson:"validity"`  // "valid", "invalid", "expired"
    CreatedAt            *MongoTime         `bson:"created_at"`
    UpdatedAt            *MongoTime         `bson:"updated_at"`

    // Token fields
    AccessToken          string             `bson:"access_token,omitempty"`
    RefreshToken         string             `bson:"refresh_token,omitempty"`
    LongAccessToken      string             `bson:"long_access_token,omitempty"`      // Facebook
    OAuthToken           string             `bson:"oauth_token,omitempty"`            // Twitter
    OAuthTokenSecret     string             `bson:"oauth_token_secret,omitempty"`     // Twitter
    TokenExpiresAt       *MongoTime         `bson:"token_expires_at,omitempty"`
    TokenIssuedAt        *MongoTime         `bson:"token_issued_at,omitempty"`
    ExpiresIn            int64              `bson:"expires_in,omitempty"`
    RefreshExpiresIn     int64              `bson:"refresh_expires_in,omitempty"`
    Scope                string             `bson:"scope,omitempty"`

    // Analytics timestamps
    LastAnalyticsUpdatedAt           *MongoTime `bson:"last_analytics_updated_at,omitempty"`
    LastInsightsAnalyticsUpdatedAt   *MongoTime `bson:"last_insights_analytics_updated_at,omitempty"`
    LastFansAnalyticsUpdatedAt       *MongoTime `bson:"last_fans_analytics_updated_at,omitempty"`      // Facebook
    LastVideoAnalyticsUpdatedAt      *MongoTime `bson:"last_video_analytics_updated_at,omitempty"`     // Facebook
    LastGroupAnalyticsUpdatedAt      *MongoTime `bson:"last_group_analytics_updated_at,omitempty"`     // Facebook
    LastLinkPreviewUpdatedAt         *MongoTime `bson:"last_link_preview_updated_at,omitempty"`        // LinkedIn

    // Platform-specific fields
    FanCount             int                `bson:"fan_count,omitempty"`         // Facebook
    Permissions          []string           `bson:"permissions,omitempty"`       // Facebook
    PostedAs             interface{}        `bson:"posted_as,omitempty"`         // Facebook
    UserDetails          interface{}        `bson:"user_details,omitempty"`      // Instagram
    IsBusiness           bool               `bson:"is_business,omitempty"`       // Instagram
    Username             string             `bson:"username,omitempty"`          // Instagram/Twitter/Pinterest
    ScreenName           string             `bson:"screen_name,omitempty"`       // Twitter
    Headline             string             `bson:"headline,omitempty"`          // LinkedIn
    LinkedinProfileID    string             `bson:"linkedin_profile_id,omitempty"` // LinkedIn URN
    Verified             bool               `bson:"verified,omitempty"`          // Twitter
    VerifiedType         string             `bson:"verified_type,omitempty"`     // Twitter
    SubscriptionType     string             `bson:"subscription_type,omitempty"` // Twitter API tier
    DeveloperAppID       string             `bson:"developer_app_id,omitempty"`  // Twitter
    APIKey               string             `bson:"api_key,omitempty"`           // Twitter
    APISecret            string             `bson:"api_secret,omitempty"`        // Twitter
    MetaData             interface{}        `bson:"meta_data,omitempty"`         // GMB
    Locality             string             `bson:"locality,omitempty"`          // GMB
    PostalCode           string             `bson:"postal_code,omitempty"`       // GMB
    RegionCode           string             `bson:"region_code,omitempty"`       // GMB
    LanguageCode         string             `bson:"language_code,omitempty"`     // GMB
    ProfileID            string             `bson:"profile_id,omitempty"`        // Pinterest
    BoardID              string             `bson:"board_id,omitempty"`          // Pinterest
    LinkedTo             primitive.ObjectID `bson:"linked_to,omitempty"`         // Pinterest boards linked to profiles
    URL                  string             `bson:"url,omitempty"`               // Pinterest

    // Scheduling
    QueueSlots           []interface{}      `bson:"QueueSlots,omitempty"`

    // Validity tracking
    InvalidTries         int                `bson:"invalid_tries,omitempty"`
    SentInvalidEmail     int                `bson:"sent_invalid_email,omitempty"`
    ValidityError        string             `bson:"validity_error,omitempty"`
    ValidityStatus       int                `bson:"validity_status,omitempty"`
    LimitExceedTries     int                `bson:"limit_exceed_tries,omitempty"`

    // External connections
    ConnectionViaLink    bool               `bson:"connection_via_link,omitempty"`
    ConnectionLinkID     primitive.ObjectID `bson:"connection_link_id,omitempty"`

    // Preferences and extra data
```

### 2. Repository Layer (`/src/internal/repository/mongodb/`)

#### Current Implementation

- Separate repositories: `FacebookAccountRepository`, `InstagramAccountRepository`, `LinkedinAccountRepository`
- Platform-specific collection names hardcoded
- Duplicate methods across repositories

#### New Unified Repository (Clean Implementation)

```go
// UnifiedSocialRepository handles all social platform accounts
type UnifiedSocialRepository interface {
    // Core methods
    FindByID(ctx context.Context, id primitive.ObjectID) (*SocialIntegration, error)
    GetByPlatformID(ctx context.Context, platformType, platformID string) (*SocialIntegration, error)
    GetValidAccounts(ctx context.Context, platformType, accountType string) ([]SocialIntegration, error)
    GetAccountsByWorkspace(ctx context.Context, workspaceID primitive.ObjectID, platforms []string) ([]SocialIntegration, error)
    GetAccountsNeedingUpdate(ctx context.Context, platformType string, lastUpdateField string, hours int) ([]SocialIntegration, error)

    // Update methods
    Update(ctx context.Context, id primitive.ObjectID, updates primitive.M) error
    UpdateAnalyticsTimestamp(ctx context.Context, id primitive.ObjectID, timestampType string, timestamp time.Time) error
    UpdateTokens(ctx context.Context, id primitive.ObjectID, tokens map[string]string) error
    UpdateState(ctx context.Context, id primitive.ObjectID, newState string) error
    UpdateValidity(ctx context.Context, id primitive.ObjectID, newValidity string) error

    // Create and Delete
    Create(ctx context.Context, account *SocialIntegration) (primitive.ObjectID, error)
    Delete(ctx context.Context, id primitive.ObjectID) error
}
```

## Field Mapping Reference

### Unified Field Structure

All platforms use the same core fields:

| Field                 | Purpose                     | Example                                   |
| --------------------- | --------------------------- | ----------------------------------------- |
| `platform_type`       | Platform identifier         | `"facebook"`, `"instagram"`, `"linkedin"` |
| `platform_identifier` | Platform-specific unique ID | Facebook page ID, Instagram user ID, etc. |
| `platform_name`       | Display name                | Page/Profile/Account name                 |
| `platform_logo`       | Profile image URL           | Avatar/logo URL                           |
| `platform_url`        | Profile URL                 | Link to social profile                    |
| `state`               | Account state               | `"Added"`, `"Deleted"`, `"Disabled"`      |
| `validity`            | Token status                | `"valid"`, `"invalid"`, `"expired"`       |

### Platform-Specific Field Usage

#### Facebook

- `platform_identifier` = Facebook Page/Group ID
- `long_access_token` = Page long-lived token
- `fan_count` = Number of page fans
- `permissions` = Granted FB permissions array

#### Instagram

- `platform_identifier` = Instagram account ID
- `user_details` = Nested object with profile info
- `is_business` = Business account flag
- Token may be in `user_details.access_token` (encrypted)

#### LinkedIn

- `platform_identifier` = LinkedIn ID
- `linkedin_profile_id` = LinkedIn URN (if different)
- `headline` = Profile headline

#### Google My Business

- `platform_identifier` = Location ID (was `name` field)
- `platform_name` = Location name (was `location_name`)
- `platform_logo` = Location image (was `account_image`)
- `meta_data` = Location metadata

## Query Examples

### Before (Multiple Collections)

```go
// Facebook accounts
fbRepo := mongodb.NewFacebookAccountRepository(db, logger)
fbAccounts, _ := fbRepo.GetValidAccounts(ctx, "Page")

// Instagram accounts
igRepo := mongodb.NewInstagramAccountRepository(db, logger)
igAccounts, _ := igRepo.GetValidAccounts(ctx)

// LinkedIn accounts
liRepo := mongodb.NewLinkedInAccountRepository(db, logger)
liAccounts, _ := liRepo.GetValidAccounts(ctx)
```

### After (Unified Collection)

```go
// Single repository for all platforms
unifiedRepo := mongodb.NewUnifiedSocialRepository(db, logger)

// Facebook accounts
fbAccounts, _ := unifiedRepo.GetValidAccounts(ctx, "facebook", "Page")

// Instagram accounts
igAccounts, _ := unifiedRepo.GetValidAccounts(ctx, "instagram", "")

// All accounts for a workspace
allAccounts, _ := unifiedRepo.GetAccountsByWorkspace(ctx, workspaceID,
    []string{"facebook", "instagram", "linkedin"})
```

## Considerations and Best Practices

### Key Implementation Notes

1. **Token Decryption**

   - Tokens remain encrypted in the same format
   - Instagram may have nested token in `user_details.access_token`
   - Decryption logic stays unchanged

2. **Performance Optimization**

   - Ensure index on `platform_type` field for efficient filtering
   - Use compound index on `[platform_type, validity, state]` for common queries
   - Batch operations when fetching multiple platforms

3. **Data Validation**

   - Always validate `platform_type` before queries
   - Check for required fields based on platform
   - Handle missing or null fields gracefully

4. **Error Handling**
   - Log platform type with all errors for easier debugging
   - Include account ID and platform identifier in error messages
   - Implement proper retry logic for transient failures
